# Behavioral Contract: EZ404

DN404 hybrid (ERC-20 + ERC-721 mirror) + dual-currency coin-age reward ledger + capped public mint.

## Token surface (DN404)
- Standard ERC-20 (`transfer`/`approve`/…) and the DN404 mirror ERC-721.
- `_unit() = 10_000e18` (override).
- `name = "EZ404"`, `symbol = "EZ"` (placeholder — confirm before mainnet).

## Mint
- `publicMint(uint256 qty) payable`
  - `require(msg.value == qty * pbMintPrice)`
  - `require(mintedUnits + qty <= MAX_SUPPLY)`
  - mints `qty * _unit()` EZ404 to `msg.sender` (settles rewards first)
  - routes mint ETH to the locked pool seed (v1: 100%; see "% to LP" knob, D-9)

## Reward intake (hook only)
- `notifyFeeETH() payable onlyHook` → `_accrue(ETH, msg.value)`
- `notifyFeeToken(uint256 amt) onlyHook` → `_accrue(token, amt)` (tokens already transferred in)

## Claim
- `claim()` — settles `msg.sender`, pays `claimable0` (ETH) and `claimable1` (EZ404), zeroes both.

## Seed mint (hook only)
- `mintForSeed(address to, uint256 amount) onlyHook` — mints the pool side of the seed (to the
  PoolManager). `to` must be excluded so it never accrues.

## Wiring (controller only, one-time each)
- `setHook(address)` — resolves the token↔hook ctor chicken-and-egg; settable once.
- `setExcluded(address, bool)` — mark PoolManager / hook / token / dead / pool as non-accruing.

## Invariants
- **INV-1** Every balance-changing path calls `_syncRewards` *before* the balance moves.
- **INV-2** Excluded accounts have zero `claimable*` and never contribute to `B`/`S`.
- **INV-3** `Σ claimed + Σ claimable + undist0/1 + dust == Σ notified` per currency (no over-pay).
- **INV-4** `onlyHook` functions revert for any caller other than the wired hook.
- **INV-5** A balance with age ≈ 0 earns ≈ 0 from a fee distributed in the same block (JIT-proof).
